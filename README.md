# Навигатор по смыслу. Семантический поиск по коду.

## Структура проекта

```
semantic-search-case/
│
├── data/                        # Данные, которые будут анализироваться
│
├── notebooks/                   # Рабочие ноутбуки - каждый работает в своём
│
├── src/                         # Общий код. Функции, которые используются во всех ноутбуках
│
├── api/                         # FastAPI-сервис
│
├── final_notebook.ipynb         # Итоговый ноутбук для защиты
├── report.md                    # Отчёт с выводами и нашей рекомендациеq
├── pyproject.toml               # Конфигурация проекта, список зависимостей (управляется uv)
├── uv.lock                      # Точные версии всех пакетов
├── .python-version              # Версия Python для проекта
└── requirements.txt             # Сгенерированный файл (требование задания)
```

## Быстрый старт

### Шаг 1. Установи uv

`uv` это современный менеджер пакетов для Python, фактически стандарт индустрии.

**Linux:**
```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

**Windows** (в PowerShell):
```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

После установки перезапусти терминал и проверь:
```
uv --version
```


### Шаг 2. Клонируй репозиторий

```bash
git clone https://github.com/Eugeniouo/semantic-search-case.git
cd semantic-search-case
```


### Шаг 3. Подтяни зависимости

Одной командой устанави все зависимости из `uv.lock`:
```bash
uv sync
```

Создаст папку `.venv` внутри проекта, это изолированное окружение.


### Шаг 4. Запусти JupyterLab

```bash
uv run jupyter lab
```
Запустит jupyter в нужном окружение.

## Как добавить новый пакет?

Если нужна новая библиотека - не устанавливаем через `pip install`.
Пишем `uv add`, чтобы все в команде получили обновления:

```bash
uv add <название-пакета>
```

После этого обязательно обнови `requirements.txt`.
```bash
uv export --no-hashes > requirements.txt
```
